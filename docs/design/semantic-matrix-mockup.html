交互说明
1分层两视图。矩阵只放实体与属性 （跨表映射是它的本质形状）；指标是公式组合而非跨表映射，独立成结构化配置视图。 矩阵定义"词汇表"（属性、语义角色），指标引用词汇表来组合。
2指标八要素。指标名（含别名）、 指标描述、关联的数据表（单表，分子分母都在其上计算）、统计时间的语义层字段 （引用矩阵中的语义角色，漂移冻结沿"物理列 → 语义角色 → 指标"传导）、 filter 表达式（公共过滤，同时作用于分子与分母，如排除测试账号；可留空）、 分子 + 分子描述、分母 + 分母描述（分母留空即单一聚合指标，如 GMV）。 生成 SQL 形如：AGG(分子) / AGG(分母) FROM 关联表 WHERE filter AND 时间范围。 保存前强制试算预览。
•模型与配置分层。底层数据模型中， 分子与分母是独立组件，各自携带 表达式 / 描述 / 数据表 / filter（天然支持跨表比率与 分侧过滤）；配置界面当前只开放指标级的单一数据表与单一 filter，保存时写入两个组件。 未来放开跨表只是解锁 UI 字段，无需改模型与存储。
•时间口径一致性（硬约束）。 统计时间的语义层字段永远是指标级的单一字段——分子分母必须共用同一时间口径， 这正是它不下放到组件级的原因。当分子分母来自不同表时，该语义角色必须在 两张表都有绑定（矩阵中同一行的两个格都非空）；保存与试算时强制校验， 缺失则拒存并提示"请先在映射矩阵为表 X 绑定语义角色 Y"。生成 SQL 时各表用 各自绑定的物理列（含 SQL 转换表达式）过滤同一时间范围。
3单元格 = 绑定 + 证据。绿勾表示人工判决过； 橙色菱形是算法证据（值域重叠 / 历史 join），点击徽章可一键转为已确认，等价于确认队列里的选择题。
4空格点击就地绑定，两种模式。 "选择列"模式列出该表字段并按证据分排序推荐；"ƒ SQL 转换"模式输入表达式片段 （只允许引用本表字段），保存前经护栏做方言解析校验，并用采样行实时预览转换结果—— 示例中"注册年份"由 strftime('%Y', signup_dt) 派生。已保存的转换绑定在单元格中 以 ƒ 前缀展示（如"支付月份"行）。
5列 = 已接入数据表。"添加数据表列" 接元数据浏览器，勾选新表后它成为矩阵新列；超过 4 列时矩阵横向滚动，左列冻结。
6状态体系与后端一一对应。已确认 = verified，草稿 = 冷启动/LLM 产物，冻结 = schema 漂移；徽章即数据，无需新增模型字段。
